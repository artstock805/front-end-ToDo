# Todo 관리 스킬

로컬 Todo 앱(`C:\Users\ssei8\todo-app`)의 SQLite DB를 `todo-app` MCP 서버를 통해 직접 관리합니다.
여기서 변경한 내용은 브라우저 앱(http://localhost:3456)에서도 동일하게 보입니다.

## 사용할 수 있는 MCP 도구

| 도구 | 용도 | 주요 인자 |
|------|------|-----------|
| `mcp__todo-app__list_todos` | 목록 조회 | `q`(검색어), `tag`(태그명), `today`(오늘 마감만) — 모두 선택 |
| `mcp__todo-app__add_todo` | 할 일 추가 | `title`(필수), `due_date`(YYYY-MM-DD), `tags`(문자열 배열) |
| `mcp__todo-app__complete_todo` | 완료/되돌리기 | `id`(필수), `done`(false면 미완료로 되돌림) |
| `mcp__todo-app__delete_todo` | 삭제 | `id`(필수) |
| `mcp__todo-app__list_tags` | 태그 목록 | (없음) |

## 작업 방식

### 할 일 추가
- 사용자 문장에서 제목을 뽑아 `add_todo`를 호출합니다.
- **날짜 해석**: "오늘"→오늘 날짜, "내일"→+1일, "모레"→+2일, "이번 주 금요일" 등도 실제
  `YYYY-MM-DD`로 변환해 `due_date`에 넣습니다. 오늘 날짜는 세션 컨텍스트의 currentDate를 기준으로 합니다.
- 분야/맥락이 드러나면 태그로 넣습니다. 예: "학교 과제" → `tags: ["학교", "과제"]`.
  단, 억지로 태그를 만들지 말고 자연스러울 때만 붙입니다.

### 목록 조회
- "할 일 보여줘"→ 조건 없이 `list_todos`.
- "오늘 뭐 해야 해?"→ `today: true`.
- "○○ 관련 할 일"→ 먼저 `list_tags`로 태그를 확인하고 `tag` 또는 `q`로 조회.

### 완료 처리 / 삭제
- 사용자가 "끝냈어", "다 했어"라고 하면 목록에서 해당 항목의 `id`를 찾아 `complete_todo`.
- 어떤 항목인지 모호하면 먼저 `list_todos`로 확인하고, 후보가 여러 개면 사용자에게 되묻습니다.
- **삭제는 되돌릴 수 없으므로**, 대상이 명확하지 않으면 반드시 확인 후 `delete_todo`를 호출합니다.
  "완료"와 "삭제"를 혼동하지 마세요 — 다 한 일은 보통 완료(complete)이지 삭제가 아닙니다.

## 주의
- id는 목록 조회 결과에 `#숫자`로 표시됩니다. 이 숫자를 도구의 `id` 인자로 씁니다.
- 도구 응답은 이미 사람이 읽기 좋은 형태이므로, 그대로 요약해 전달하면 됩니다.
- 웹서버가 꺼져 있어도 이 도구들은 동작합니다(DB 직접 접근). 별도로 서버를 켤 필요 없습니다.
